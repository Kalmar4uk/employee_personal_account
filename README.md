В проекте используется:
* Python 3.11.9
* Django 5.2.5
* Django rest framework 3.16.1

## Как запустить проект

*Клонировать репозиторий*
```
https://github.com/Kalmar4uk/dashboard_project.git
```
*В корневой директории проекта создать файл .env*
```
SECRET_KEY_FOR_REQUEST=Необходимо сгенерировать секретный ключ, данный ключ используется для запросов телеграм бота
SECRET_KEY_DJANGO=Необходимо сгенерировать секретный ключ
POSTGRES_USER=пользвоатель для БД
POSTGRES_PASSWORD=пароль для БД
POSTGRES_DB=название БД
DB_HOST=db
DB_PORT=5432
DSN=Ссылка для отслеживания ошибок в Sentry
DEBUG=True
```

1. По умолчанию при локальном запуске в проекте будет использована SqLite3
2. Секретные ключи необходимо сгенерировать до запуска проекта, так как без ```SECRET_KEY_DJANGO``` могут быть ошибки при запуске

> Если проект не планируется запускать локально из контейнеров, то переменные POSTRES_* и DB_* нет необходимости выставлять

* При локальном запуске через ```python manage.py runserver``` необходимо создать и активировать виртуальное окружение и выполнить миграции.
* Так же в проекте есть ```docker compose``` для запуска в контейнерах, открыть Docker Desktop и находясь в директории с файлом прописать команду ```docker compose up -d```, после чего выполнить миграции внутри запущенного контейнера.


### Локальный запуск:

*Находясь в корневой директории проекта создать и активировать виртуальное окружение*
```
python -m venv venv
```

*Обновить pip*
```
python -m pip install --upgrade pip
```

*Установить зависимости*
```
pip install -r requirements.txt
```

*Перейти в директорию с файлом ```manage.py``` и выполнить миграции*
```
python manage.py migrate
```

### Запуск в контейнерах docker

*Находясь в корневой директории проекта прописать команду*

```
docker compose up -d или docker compose up
```

*После запуска контейнеров необходимо выполнить миграции внутри backend контейнера*

```
docker compose exec backend python manage.py migrate
```

> Вне зависимости от того, как запущен проект он станет доступен по ссылке http://localhost:8000/
